A "set it and forget it" CLI tool for automated MySQL backups on Ubuntu. This script handles credential management, scheduling, and automatic 30-day cleanup without the bloat.🚀 Quick StartDownload/Create the script:Save the code into a file named auto_backup.sh.Make it executable:Bashchmod +x auto_backup.sh
Run the Wizard:Bash./auto_backup.sh
✨ Key FeaturesCredential Wizard: Validates your DB connection before saving.Secure Storage: Stores credentials in ~/.sentinel_db.conf with 600 permissions (owner-only).Cron Integration: Automatically appends/updates its own entry in your system crontab.Auto-Cleanup: Automatically deletes backups older than 30 days after every successful run.Silent Backups: Uses a temporary defaults file to bypass "Enter Password" prompts, making it 100% compatible with automation.🛠 Menu OptionsOptionDescription1) Setup CredentialsEnter host, user, pass, and DB name. It tests the connection live.2) Manual BackupTriggers an immediate backup and runs the 30-day cleanup logic.3) Configure CronChoose between Daily or Weekly schedules. It manages the system crontab for you.4) UninstallWipes the config file and removes all Sentinel-DB cron jobs.📂 File LocationsBackups: Saved to ~/db_backups/Config: Hidden at ~/.sentinel_db.confLogs: Errors are printed directly to the console or captured by system cron logs.Note: This script is optimized for Bash. If you run it using sh ./auto_backup.sh, it will fail due to syntax differences. Always use ./auto_backup.sh or bash auto_backup.sh.
